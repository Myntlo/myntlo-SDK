# Security Policy

## Supported Versions

`@myntlo/sdk` is currently pre-1.0 (`0.1.0`). Only the latest published version on npm is supported with security fixes. Please always upgrade to the latest release before reporting an issue.

## Reporting a Vulnerability

If you discover a security vulnerability in this SDK, please report it privately using [GitHub Security Advisories](https://github.com/Myntlo/myntlo-SDK/security/advisories/new).

Please do not open a public GitHub issue for security vulnerabilities. Public issues are visible to everyone and could put users at risk before a fix is available.

When reporting, please include as much detail as possible, such as:

- A description of the vulnerability and its potential impact
- Steps to reproduce, including affected SDK version(s)
- Any relevant code samples, logs, or proof-of-concept

## Response Expectations

We aim to acknowledge new reports within a few business days. Response and fix timelines may vary depending on the complexity and severity of the issue. We will keep you updated as we investigate and work toward a resolution.

Thank you for helping keep Myntlo and its users safe.
